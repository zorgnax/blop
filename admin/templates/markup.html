Edit Help

The posting format is similar to Markdown, except square brackets "[]" are used more like BBCode.

  • Most importantly, HTML is passed through untouched.

    <a href="http://google.com">Okay!</a>
    		 
    <a href="http://google.com">Okay!</a>
    		 Okay!

    If you add the markup="1" attribute, the text of the element will be parsed as more markup.

    <div markup="1">**Bold** Text</div>
    		 
    <div ><p><b>Bold</b> Text</p></div>

    Bold Text

  • Blocks of text become paragraphs.

    This is a normal paragraph.
Single line breaks are included

Another paragraph.
    		 
    <p>This is a normal paragraph.
Single line breaks are included</p>

<p>Another paragraph.</p>

    This is a normal paragraph. Single line breaks are included

    Another paragraph.

  • Indented lines are preformatted text. HTML characters are encoded automatically.

    This is a normal paragraph.

    a < b
    		 
    <p>This is a normal paragraph.</p>

<pre><code>a &lt; b</code></pre>

    This is a normal paragraph.

    a < b

  • Lines starting with ">" are block quoted.

    This is a normal paragraph.

> A quote.

He said:
> She said:
>> what
>> foo?

    <p>This is a normal paragraph.</p>

<blockquote><p>A quote.</p></blockquote>

<p>He said:</p>
<blockquote><p>She said:</p>
<blockquote><p>what
foo?</p></blockquote></blockquote>

    This is a normal paragraph.

    A quote.

    He said:

    She said:

    what foo?

  • Headings are underlined or start with hash marks.

    Foo
===

Bar
---

# Baz

## Qux
    		 
    <h1>Foo</h1>

<h2>Bar</h2>

<h1>Baz</h1>

<h2>Qux</h2>

    Foo

    Bar

    Baz

    Qux

  • Bold text can be surrounded by "**". Unlike Markdown, you cannot use "__".

    **Bold Text**
    		 
    <b>Bold Text</b>
    		 Bold Text

  • Italicized text can be surrounded by "*". Unlike Markdown, you cannot use "_".

    *Italicized Text*
    		 
    <i>Italicized Text</i>
    		 Italicized Text

  • Backticks can make text look like code examples. Multiple backticks can be used to surround inner backticks.

    lol `tdm`.
    		 
    lol <code>tdm</code>.
    		 lol tdm.

  • Backslashes can be used to escape characters that would otherwise cause markup to change.

    \*\*Not Bolded\*\*
    		 
    **Not Bolded**
    		 **Not Bolded**

  • Three or more astericks or dashes result in a horizontal rule.

    * * *

---
    		 
    <hr/>

<hr/>

  • Text starting with http:// are automatically converted to links.

    http://a.co
    		 
    <a href="http://a.co">http://a.co</a>
    		 http://a.co

  • Lists can be any block of text that starts with a "*", "+", "-", or a digit followed by ".". If the item content is more than one line. It will be surrounded by <p> tags.

    1. foo
2. bar
    * items
    * can
    * nest
3. baz

    <ol>
<li>foo</li>
<li>bar
<ul>
<li>items</li>
<li>can</li>
<li>nest</li>
</ul>

</li>
<li>baz</li>
</ol>
    1. foo
    2. bar
      • items
      • can
      • nest
    3. baz

  • Square bracket commands are for special tasks. There can be any number of square bracket commands depending on the plugins you have installed. If the command is unrecognized, the text will be unmodified.

    [link a.jpg /]

[unrecognized]

    <p><a href="./mu/files/a.jpg">a.jpg</a></p>

<p>[unrecognized]</p>

    a.jpg

    [unrecognized]

Built-in Square Bracket Commands

  • [code] will show its content as a code sample. You can put text after the tag name to escape inner [code][/code] spans like this [code-ex1][code][/code][/code-ex1]. You can even give it a lang attribute, which can be used for syntax highlighting. Otherwise, the result is exactly like an indented block or backticks. In fact, if it's alone in a paragraph, the result gets surrounded by <pre> tags:

    [code lang="python"]
def average(values):
    """Computes the arithmetic
    mean of a list of numbers.

    >>> print average([20, 30, 70])
    40.0
    """
    return sum(values, 0.0) / len(values)
[/code]

    def average(values):
    """Computes the arithmetic
    mean of a list of numbers.

    >>> print average([20, 30, 70])
    40.0
    """
    return sum(values, 0.0) / len(values)

  • [link] is a shortcut for adding links. If the link relative like "image.jpg", the URL will be based in the post's content directory. i.e. /post/24/image.jpg:

    [link a.jpg "opt title"]Click[/link]
    		 Click

  • [gallery] will show a list of thumbnail links to the images attached to the post. The size attribute can be used to set the size of the thumbnails, which can be either "small", "medium", "large", "nxn", or "xn>" (full spec is at http://www.imagemagick.org/script/command-line-processing.php#geometry).

    [gallery size="x70>" /]

    The x70> above sets the height to 70 pixels but allows the width to change to maintain proportions of the original image.

  • [thumb] will show one picture as a clickable thumbnail. The thumbnail size can be set with the size attribute just as [gallery]. The name of the picture is the first (unnamed) argument. It needs to be found in the post's (or page's) content directory (usually /post/1234/).

    [thumb a.jpg size="small" /]

  • [image] will show an image from the post (or page).

    [image "j.jpg" /]

  • [listing] will show a list of links to files attached to the post.

    [listing /]
    • a.jpg 5M
    • b.jpg 265.9K
    • c.jpg 190.9K
    • d.jpg 523.6K
    • e.jpg 321.8K
    • f.jpg 321K
    • g.jpg 174.9K
    • h.jpg 95.3K
    • i.jpg 208.7K
    • j.jpg 37.9K
